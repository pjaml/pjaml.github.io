How to use MSI

Peter Lutz

Created: 2025-03-30 Sun 02:12

So you’ve written some code…

Suppose you have a script that takes way too long to run on your own computer. You want to run it on the MSI supercomputer and take advantage of their computing resources.

This can be a challenge because…

  • You may need to set up an account on MSI if you don’t have access yet
  • You’re not familiar with the terminal and command-line shells
  • You don’t know how to submit and manage jobs using the MSI scheduler (Slurm)

Why can’t I just run my script?

Even when you’re connected to MSI and running an interactive session, you can’t just run your script, e.g. with 

lutzx119@ahl01 [~] % Rscript myscript.R

Well you can, but this doesn’t take advantage of MSI’s computing power.

You need to run a job via the scheduler

MSI uses a scheduler — software that allows researchers to request time and resources and efficiently manages all the submitted jobs.

The scheduler MSI uses is called Slurm, and you communicate with it through various commands that start with “s”: srun, sbatch, squeue, scancel, etc.

Understanding MSI and Slurm isn’t absolutely necessary

You could just submit your job via Slurm and hope that it works…

…at least you wouldn’t be running the script for hours on your own laptop.

A lot of people seem to use MSI this way.

But if you understand how to efficiently request resources and communicate with Slurm, you can often get your script to run much faster. 💪😎

Learn Slurm terms

The main MSI system is called Agate.

In Slurm, it’s called a cluster — think of it as a bunch of computers that are loosely connected together.

Each computer in the cluster is called a node. Agate has hundreds of nodes, each with different connected resources (memory, GPUs, etc.).
